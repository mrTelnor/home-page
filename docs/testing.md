# Гайд для тестировщика

Документ для проверки функционала **Home Page** (telnor.ru) и Telegram-бота `@volkov_homepage_bot`.

## 0. Подготовка

### Получить аккаунт

1. Запросить у админа **инвайт-код** для регистрации
2. Открыть [telnor.ru](https://telnor.ru) → Войти → Зарегистрироваться
3. Ввести логин, пароль (≥8 символов), инвайт-код
4. После регистрации — вход через логин/пароль

### Привязать Telegram (для тестирования бота)

1. На сайте: Профиль → раздел Telegram → нажать «Login with Telegram»
2. Откроется виджет Telegram → подтвердить
3. После привязки в Telegram открыть бота `@volkov_homepage_bot` → `/start`

### Где смотреть логи (для админов / разработчиков)

```bash
ssh -p 9922 -i ~/.ssh/GitHub_SSH telnor@147.45.183.98 'docker logs <container> --tail 50'
```

Контейнеры: `backend`, `frontend`, `bot`, `cron`, `postgres`, `traefik`.

Поток логов в реальном времени: добавь `-f` (`docker logs bot -f --tail 30`).

## 1. Web — авторизация и доступ

### Регистрация
- Без инвайт-кода → 403, ошибка «Неверный инвайт-код»
- С коротким паролем (<8 символов) → 422, валидация формы
- С существующим username → 409, «Имя пользователя занято»
- Регистр username игнорируется: `Ivan` и `ivan` — один и тот же пользователь

### Вход
- Корректные логин/пароль → редирект на главную, появляется имя пользователя в шапке
- Неверный пароль → 401, «Неверный логин или пароль»
- Неизвестный логин → 401

### Гостевой вход
- На странице `/login` — кнопка «Войти как гость» → попадаешь на `/recipes` без авторизации
- В шапке у гостя нет блока «Профиль / Выйти», только «Войти»
- Гость **может** открывать список рецептов и детали отдельного рецепта
- Гость **не может**: голосовать, добавлять рецепты, редактировать чужие, видеть меню дня и историю голосований (редирект на login)

### Выход
- Кнопка «Выйти» в шапке → cookie удаляется → редирект на `/login`

## 2. Web — рецепты

### Список (`/recipes`)
- Грид карточек с FoodGlyph-иконкой (15 типов × 10 палитр)
- Если у рецепта не задан `glyph_kind`/`glyph_color` — иконка выбирается детерминированно по хешу названия
- Сортировка: по алфавиту / дате добавления / дате изменения; направление ↑ / ↓
- Сортировка сохраняется в `localStorage` — после перезагрузки страницы выбор остаётся
- Описание на карточке — **в одну строку** с многоточием (`truncate`)
- Под описанием — счётчики «N порц., M ингр.»
- Кнопка «Добавить рецепт» видна только авторизованным

### Создание (`/recipes/new`)
- Поля: Название (≥3 симв., обязательное), Порций, Иконка (тип + цвет), Ингредиенты, Описание
- В блоке Иконка — превью FoodGlyph рядом с пикером (тип через select, цвет — кружки-swatches)
- При выборе «Авто» (↺) — иконка определяется по названию
- Хотя бы один ингредиент с названием и количеством — обязателен
- Если задана единица измерения, в поле «Кол-во» должно быть число (иначе ошибка)
- После сохранения → редирект на детали рецепта

### Просмотр (`/recipes/:id`)
- Hero-блок с FoodGlyph
- Хлебная крошка `← К списку рецептов` (всегда работает, не зависит от истории браузера)
- Метрики: время добавления, изменения
- Ингредиенты с пересчётом порций — поле «Порций» меняет количество ингредиентов пропорционально
- Кнопка «Сбросить» возвращает к исходному количеству порций (серая, если уже исходное)
- Кнопки «Редактировать» / «Удалить» видны только автору рецепта или админу
- Удаление спрашивает подтверждение
- Удаление рецепта, который сейчас в активном голосовании → 409

### Редактирование (`/recipes/:id/edit`)
- Хлебная крошка `← К рецепту`
- Все поля заполнены текущими значениями, в т.ч. иконка
- Чужой рецепт пытается редактировать обычный пользователь → 403
- Админ может редактировать любой

### Гостевой просмотр
- Открыть `/recipes` без логина → видишь список без кнопок «Добавить» / «Редактировать»
- Открыть `/recipes/:id` без логина → видишь детали без кнопок управления

## 3. Web — голосование

### Цикл (автоматический по cron, GMT+3)

| Время | Действие | Видно в UI |
|---|---|---|
| 08:00 | Создание меню + 3 случайных рецепта (статус `collecting`) | На главной — «Сбор предложений» |
| 13:00 | Финализация (статус `voting`) | «Голосование открыто» |
| 17:00 | Закрытие, определение победителя (статус `closed`) | «Победил: …» |

### Что проверить (`/vote`)

#### Стадия `collecting` (сбор)
- Видны 3 случайных рецепта
- Кнопка «Предложить рецепт» открывает диалог поиска
- Можно предложить **только 1** рецепт (admin — до 3)
- Предложенный рецепт нельзя добавить второй раз — 409
- Рецепт уже в меню — 409

#### Стадия `voting` (голосование)
- Inline-кнопки рядом с каждым рецептом
- После голосования — кнопка «Отменить голос» появляется на выбранном
- Под рецептом видны **аватары проголосовавших** (стек из 4 первых + «+N»)
- Счётчик «N гол.» рядом
- Можно отменить голос и переголосовать
- Повторный голос на то же → 409

#### Стадия `closed`
- Победитель помечен бейджем «Победитель»
- Карточка победителя выделена рамкой и фоном
- Аватары проголосовавших всё ещё видны

### История (`/vote/history`)
- Список меню по дням, отсортирован от новых к старым
- Клик на меню — раскрывается список рецептов с голосами

### Ручной триггер цикла (для админа/тестера)
В Swagger (`api.telnor.ru/docs`, доступ ограничен по IP) или через curl с админ-cookie:
- `POST /api/menus/create-daily` — создать меню (ошибка 409 если уже есть)
- `POST /api/menus/finalize` — открыть голосование
- `POST /api/menus/close-voting` — закрыть, определить победителя
- `DELETE /api/menus/{id}` — удалить (для пересоздания)

## 4. Web — личный кабинет

### Профиль (`/profile`)
- Имя, день рождения, пол, фамилия Волков/Волкова
- При установке `is_volkov=true` + `gender=female` фамилия отображается как «Волкова»
- Сохранение — кнопка «Сохранить»; повторный клик до изменения формы не делает запрос

### Смена пароля
- Открывается в модалке
- Старый пароль должен совпадать (иначе 401)
- Новый пароль — ≥8 символов

### Telegram
- Если не привязан → виджет «Login with Telegram»
- Если привязан → бейдж «Привязан», ID, кнопка «Отвязать»
- Привязка одного и того же tg_id к разным пользователям → 409

### Уведомления
- Чекбокс «Получать уведомления от бота» — управляет `notifications_enabled`
- Когда выключен — бот не шлёт уведомления о меню/голосовании

## 5. Web — UI / тема

### Светлая / тёмная тема
- Переключатель ☾ / ☀ в шапке справа
- Выбор сохраняется в `localStorage`
- При первом заходе подхватывается `prefers-color-scheme` из ОС
- Не должно быть «вспышки белого фона» при загрузке (тема применяется до первого рендера)

### Шапка (для авторизованных)
- Логотип-волк + «Волковы» (на desktop)
- Кнопка «← Назад» (disabled на главной)
- Быстрые действия (только desktop): Проголосовать / Добавить рецепт / Открыть книгу — `text-xl` без жирности
- Тема, имя пользователя (lg+), Профиль, Выйти

### Шапка (для гостя)
- Только лого + кнопка «Войти»

### Главная (`/`)
- Гость: волк-маскот, заголовок «Семейная страница Волковых», кнопки «Смотреть рецепты» / «Войти»
- Авторизованный: «Привет, {имя}!» + VoteWidget (статус сегодняшнего голосования)

### NotFound (`/что-угодно`)
- Страница 404 с ссылкой на главную

## 6. Telegram-бот

### Основные команды
| Команда | Что делает |
|---|---|
| `/start` | Приветствие; если не привязан — ссылка на сайт |
| `/help` | Список команд |
| `/menu` | Меню дня (статус, рецепты, победитель) |
| `/vote` | Голосование inline-кнопками; есть отмена голоса |
| `/suggest` | Поиск рецепта по названию → выбор → предложение |
| `/recipes` | Пагинированный список рецептов (по 10 на странице) |
| `/schedule` | Расписание на сегодня и завтра (только админам) |
| `/mute` | Отключить уведомления |
| `/unmute` | Включить уведомления |

### Что проверить

#### Не привязанный аккаунт
- Любая команда (кроме `/start` / `/help`) → «Привяжите Telegram-аккаунт на telnor.ru/profile»

#### `/menu` в каждом статусе
- `collecting` — список рецептов, статус «Сбор предложений»
- `voting` — статус «Голосование», без победителя
- `closed` — статус «Завершено», 🏆 + название победителя

#### `/vote`
- В статусе `collecting` → «Голосование ещё не открыто»
- В статусе `voting` без голоса → кнопки рецептов
- В статусе `voting` после голоса → текущий голос отмечен ✓, кнопка «❌ Отменить голос»
- После клика на рецепт → «Голос принят», сообщение обновляется
- После клика «Отменить голос» → «Голос отменён», кнопки снова без отметок
- В статусе `closed` → «Голосование уже завершено»

#### `/suggest`
- В `collecting` → «Введите название рецепта:»
- Текст с совпадениями → inline-кнопки совпавших рецептов + «Отмена»
- Текст без совпадений → «Рецепт не найден. Добавьте его на telnor.ru/recipes/new и попробуйте снова»
- Клик на рецепт → «✅ {название} добавлен в меню!» + остальным придёт уведомление «{Имя} предложил к голосованию: {Название}»
- В не-`collecting` → «Сбор предложений закрыт»

#### `/recipes`
- Список с пагинацией (по 10), кнопки ⬅️ / ➡️
- Клик на рецепт → детали (название, порции, ингредиенты, описание)

#### `/mute` / `/unmute`
- `/mute` → 🔇 «Уведомления отключены»
- `/unmute` → 🔔 «Уведомления включены»
- Соответствующее значение `notifications_enabled` обновляется в БД

#### Кнопка-меню в Telegram
- Слева от поля ввода — кнопка с пунктами всех команд
- При нажатии на пункт — команда отправляется

### Уведомления о меню (cron-driven)
- 08:00 — «🍽 Меню дня готово!» с 3 рецептами
- 13:00 — «🗳 Голосование открыто! Используйте /vote»
- 17:00 — «🎉 Голосование завершено! Победитель: …» + список с голосами

#### Ручной триггер
```bash
ssh -p 9922 -i ~/.ssh/GitHub_SSH telnor@147.45.183.98 'docker exec cron sh -c "curl -s -X POST http://bot:8080/notify -H \"X-Cron-Secret: \$CRON_SECRET\" -H \"Content-Type: application/json\" -d \"{\\\"event\\\":\\\"voting_closed\\\"}\""'
```
Заменить `voting_closed` на `menu_created` или `voting_opened` для других событий.

## 7. Bot — Google Calendar

### `/schedule` — расписание по запросу
- Только админ → дайджест на сегодня + завтра
- Не-админ → «Команда доступна только администраторам»
- Не привязан → «Привяжите Telegram-аккаунт на telnor.ru/profile»
- В дайджесте — события из всех 4 настроенных календарей (Никита, Жена, Дочь, Семья), у каждой строки `[Календарь]`

### Утренний дайджест в 09:00 GMT+3
- Автоматически приходит всем admin-пользователям
- Если в каком-то дне нет событий → «— событий нет»
- Целодневные события показываются как «весь день» вместо времени

### Часовые напоминания
- За 60 минут до начала любого события (с допуском ±4 мин)
- Текст: «⏰ Напоминание за 1 час\n\n{название}\n🗓 {время} [Календарь]»

### Custom-напоминания из события
- Если в Google Calendar у события задан конкретный reminder (например, за 30 мин), бот тоже отправит уведомление в это время
- Использует `event.reminders.overrides`, не `useDefault`

### Тест-кейсы

| Кейс | Шаги | Ожидание |
|---|---|---|
| Часовое напоминание | Создать событие через 62-65 мин | В течение 5 мин — уведомление «за 1 час» |
| Custom reminder | Создать событие через 32-34 мин с notification «за 30 мин» | Уведомление «за 30 мин» |
| Целодневное в дайджесте | Создать all-day на завтра | В дайджесте — «весь день» |
| Несколько календарей | Создать события в разных календарях | Корректные `[Календарь]` метки |
| Дедуп | Дважды вызвать `/check-calendar` подряд | Второй раз ничего не присылается |
| Отменённое событие | Создать → удалить (до часа до начала) | Уведомление **не приходит** |
| Пустой дайджест | Удалить все события на завтра | «— событий нет» |
| Команда `/schedule` | Запустить от админа | Тот же дайджест что в 9:00 |

### Ручные триггеры

```bash
# Дайджест (повторно — нужен force=true чтобы обойти дедуп)
docker exec cron sh -c 'curl -s -X POST "http://bot:8080/check-calendar?digest=true&force=true" -H "X-Cron-Secret: $CRON_SECRET"'

# Часовая проверка вне расписания
docker exec cron sh -c 'curl -s -X POST http://bot:8080/check-calendar -H "X-Cron-Secret: $CRON_SECRET"'

# Посмотреть состояние дедупа
docker exec bot cat /data/sent_reminders.json
```

## 8. Алерты от мониторинга

[HetrixTools](https://hetrixtools.com) проверяет `telnor.ru` и `api.telnor.ru/api/health` каждые 5 минут с 4 локаций. При падении сразу шлёт webhook на `bot.telnor.ru/uptime-alert?secret=...`, бот рассылает алерт в Telegram админам.

### Что проверить
- Отключить backend (`docker stop backend` от админа на ВМ) → через 5-10 мин в Telegram у админов: 🔴 **Home Page API**: DOWN
- Запустить обратно (`docker start backend`) → 🟢 **Home Page API**: UP
- В Hetrix также есть ручная кнопка «Test Alert» в Contact List

## 9. Бэкапы

Ежедневно в 03:00 GMT+3 cron-контейнер делает `pg_dump` БД, gzip, заливает на Яндекс.Диск через WebDAV в папку `/backups/`. Файлы старше 14 дней удаляются.

### Проверить вручную
```bash
ssh -p 9922 -i ~/.ssh/GitHub_SSH telnor@147.45.183.98 'docker exec cron /usr/local/bin/backup.sh'
```
После выполнения на Яндекс.Диске должен появиться файл `homepage_YYYY-MM-DD.dump.gz`.

## 10. Известные ограничения и квирки

- **DNS-кэш Telegram для `bot.telnor.ru`** — Telegram не может разрезолвить домен `telnor.ru` после долгих попыток установить webhook (DNS забанен на их резолверах). Поэтому бот работает в **polling mode** — мгновенный отклик, но через WireGuard VPN до сервера в Германии (обход блокировок РФ).
- **Гость не сохраняет состояние** — каждый раз при заходе сортировка/тема выставляются по-новому (нет аккаунта в БД).
- **Бот может отвечать с задержкой 1-2 сек** при большом количестве сообщений (polling-цикл).
- **Custom reminders** в календаре ловятся только если у события стоит `useDefault=False` и есть явный override. Дефолтные напоминания календаря бот не знает.
- **Дедуп напоминаний** хранится в Docker volume `bot_data` — переживает рестарт. Старше 7 дней удаляется автоматически.
- **Деплой** через `ansible-playbook ... --tags <service>` пересоздаёт только указанный контейнер. Без тегов — полный деплой при изменениях в `docker-compose.yml`/`.env`.
- **Dangling Docker images** — после rebuild старые образы становятся «осиротевшими». Перед каждым деплоем плейбук делает `docker image prune -f` (задача с тегом `always`). Если ошибка `No such image: sha256:...` всё-таки появилась — на ВМ запустить `docker image prune -f && docker compose up -d --build --force-recreate <service>`.

## 11. Краткий чек-лист smoke-теста

После каждого деплоя проверить за 5 минут:

1. ☐ `https://telnor.ru` — открывается, тема переключается
2. ☐ Логин рабочей учёткой
3. ☐ Список рецептов — карточки с иконками, фильтры
4. ☐ Открыть один рецепт — детали, FoodGlyph, ингредиенты пересчитываются
5. ☐ `/menu` в боте — текущий статус
6. ☐ `/schedule` от админа — дайджест приходит
7. ☐ Ручной триггер `/check-calendar?digest=true&force=true` → дайджест
8. ☐ Открыть `https://api.telnor.ru/api/health` (с домашнего IP) — `{"status":"ok"}`
9. ☐ Логи `docker logs bot --tail 20` — без ошибок
10. ☐ Логи `docker logs backend --tail 20` — без ошибок
